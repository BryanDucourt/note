# 1 吴晶晶
### 第一部分：Visual Ticket - 智能办公平台 (2022.04 - 至今)
**核心考察点：** 低代码/可视化编辑、大数据量渲染、AI 集成、Vue3 生态。

#### 1. 关于可视化流程配置器 (核心难点)
**背景：** 简历提到“独立负责可视化流程配置器的设计与实现，拖拽式交互”。
*   **提问：** 请详细描述一下你的流程配置器的数据结构（JSON Schema）是如何设计的？节点（Node）和连线（Edge）在数据层是如何关联的？
*   **追问：**
    *   在拖拽过程中，是如何处理“回环检测”（避免死循环流程）以及复杂逻辑分支的校验的？
    *   关于“撤销/重做”（Undo/Redo）功能，你是如何实现的？是基于快照（Snapshot）还是命令模式（Command Pattern）？

#### 2. 关于 Ag-Grid 与虚拟滚动
**背景：** 简历提到“引入 Ag-Grid 实现虚拟滚动，彻底解决渲染卡顿”。
*   **提问：** 为什么选择 Ag-Grid 而不是 Element-Plus 自带的 Table 或其他轻量级虚拟列表库？在配置 Ag-Grid 时，面对**不定高度的行**或**复杂的自定义单元格渲染**，你是如何保证滚动帧率维持在 60fps 的？
*   **追问：** 如果表格需要支持“列宽拖拽”+“列顺序调整”+“后端分页排序”，这套状态同步逻辑在前端是如何管理的？

#### 3. 关于 AI 智能助手集成
**背景：** 简历提到“封装 AI 智能助手对话组件，集成 OpenAI 能力”。
*   **提问：** 前端是如何接收 OpenAI 的流式响应（Stream）的？是使用了 Server-Sent Events (SSE) 还是 WebSocket？
*   **追问：** 在对话输出时，如何实现“打字机效果”？如果 AI 返回的是 Markdown 格式的代码块，你是如何进行渲染并实现代码高亮的？有没有考虑过 XSS 攻击的防护？

### 第二部分：智能随访系统 (2021.08 - 2022.03)
**核心考察点：** React 高级应用、动态表单引擎、WebSocket 状态管理。

#### 1. 关于动态表单引擎
**背景：** 简历提到“基于 React 构建可配置化问卷系统，支持拖拽生成多题型”。
*   **提问：** 你的动态表单引擎是如何处理**表单项之间的联动逻辑**（例如：选择 A 选项后，显示题目 B，否则隐藏）的？这些联动规则在 JSON Schema 中是如何定义的？
*   **追问：** 在 React 中，如果表单项非常多（例如 100+ 项），输入时会导致整个表单组件 Re-render 吗？你是如何优化渲染性能的（考察 `React.memo`, `useCallback`, 或细粒度状态管理如 Recoil/Jotai 的思想）？

#### 2. 关于 WebSocket 与 Redux 集成
**背景：** 简历提到“基于 WebSocket 搭建实时通信，集成 Redux 进行状态管理”。
*   **提问：** 当 WebSocket 推送高频消息时（例如短时间内涌入 50 条消息），直接 dispatch 到 Redux 可能会导致页面卡顿，你是如何处理的？（考察消息队列、节流、批量更新）。
*   **追问：** 关于“断线重连”和“心跳机制”，在代码层面你是如何封装的？如果用户在断网期间发送了消息，网络恢复后如何保证消息的**顺序性**和**不丢失**？

### 第三部分：叶檀财经-eos 管理系统 (2018 - 2021)
**核心考察点：** 大文件处理、Vue2/3 差异、工程化。

#### 1. 关于大文件分片上传
**背景：** 简历提到“实现大文件分片上传与断点续传”。
*   **提问：** 在计算大文件（如 1GB 视频）的 Hash 值时，为了不阻塞主线程，你是怎么做的？（考察 Web Worker 或 `requestIdleCallback`）。
*   **追问：** 如果上传过程中某个分片失败了，你的重试机制是怎样的？服务端合并分片时，如何确保文件的完整性？

#### 2. 关于 Vue2 到 Vue3 的技术演进
**背景：** 这是一个 Vue2 项目，但你现在精通 Vue3。
*   **提问：** 从架构角度看，你认为 Vue2 的 Options API 在大型后台管理系统中最大的痛点是什么？在现在的 Vue3 项目中，你如何利用 Composition API 更好地组织逻辑？（考察 `useHooks` 逻辑复用）。
