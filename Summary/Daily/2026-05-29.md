# feature_sched_exp

- 完善环境检查与系统信息采集脚本。
  - 新增 `scripts/setup/sysinfo_lib.sh`，复用只读 sysinfo 采集逻辑。
  - `scripts/setup/check_env.sh` 增加 kernel、`lscpu`、NUMA、memory、swap、Docker、Redis、memtier、perf、sysstat、cgroup、CPU governor、sched_ext 支持项输出。
  - `scripts/setup/collect_sysinfo.sh` 支持 `--output results/raw/<run_id>/sysinfo`，写入分项文本、`warnings.txt` 和 `machine_summary.json`。
  - 默认缺失命令只 WARN；增加 `--strict`，有 WARN 时最终返回非零。
  - 增加 warning：非 performance governor、swap enabled、Docker unavailable、memtier unavailable。

- 实现 Redis/memtier workload wrapper。
  - 新增 `scripts/workloads/workload_lib.sh`，统一端口解析、命令日志、JSON metadata 输出。
  - `redis_start.sh` 支持单端口和多端口、`--bind`、`--base-dir`、`--daemonize yes/no`、`--force`、`--dry-run`。
  - `redis_stop.sh` 支持多端口停止；默认不静默停止非本脚本启动的 Redis，除非传入 `--force`。
  - `redis_warmup.sh` 和 `memtier_run.sh` 补齐 memtier 参数、输出日志和 metadata。
  - `memtier_multi_instance.sh` 支持并行压测多个 Redis port，每个实例独立 log，并生成 `manifest.json`。
  - memtier wrapper 会记录 `command`、`pid`、`start_time`、`end_time`、`exit_code`；检测连接失败输出时返回非零。

- 更新 README。

  - 增加 Redis 4-instance + parallel memtier 最小运行例子。
